### 이미 '원본 이상' 달성한 것
- **지역 위험도 순위** (`regionRank`) — 원본의 "8위/2,658곳·상위 1%"를 태그(위험 상위권/안전한 편 등)까지 얹어 구현.
- **연봉 백분위** (`salaryPercentile`) — 원본엔 없는 "업종 N곳 중 M위(상위 P%)" 정보. 원본 "업종 중앙값의 16%"보다 한 단계 위.
- **기업 탐색 필터** (`/explore`) — 시도·위험등급·정렬. 원본엔 탐색 진입점 자체가 없음.
- **OG 공유 카드** (`opengraph-image.tsx`) — 위험도 카드 자동 생성 + 메타 태그. 링크 붙여넣기 시 카드 노출은 원본과 동격.

### 아직 못 한 것 (그때vs지금 / 추천2분화+배수 / 위험도사다리 / 공유 카카오 / 즐겨찾기·비교)
- ❌ **"그때 vs 지금" 입사연도 서사** (원본 L63~73) — **전무.** 원본 바이럴의 심장("6명→49명 ▲717%", "당신이 들어온 2024년보다 43명 늘었어요"). `getMonthlyStats` 시계열은 이미 있어 **UI/계산만** 필요.
- ❌ **위험도 사다리(더좋소/덜좋소)** (원본 L75~91) — **전무.** `_topRisk` 정렬 캐시(`data.ts:280`)는 있으나 자기 위치 ±근방 슬라이스 미구현.
- ⚠️ **추천 2분화 + 배수** (원본 L97~115) — `nearbyCompanies`가 **연봉순 1종**(`data.ts:189`)뿐. (a)갈아탈 곳=저위험순, (b)더받는 곳=연봉순+자사대비 배수("7.9배") 분리 미구현. 데이터 접근은 동일, 정렬키·배수 계산만 추가.
- ❌ **공유 카카오/복사 버튼** — OG 카드는 있으나 **상세 화면에 공유 액션 버튼이 전무**(`components/` 내 ShareButton 없음, grep 확인). 원본은 헤더·캡슐에 "카카오톡/복사"(L25,73). 확산 트리거가 0.
- ❌ **즐겨찾기 / 최근 본 / 비교** — 원본에도 없는 초과 영역이나 round2-09 T2 제안 대비 미착수.

### 데이터 신뢰·카피 일관성
- ✅ 기준월: 홈에 `기준월 {DATA_YM}` 노출(`page.tsx:9`). 차트엔 시계열 범위 표시.
- ⚠️ **회사 상세엔 기준월 표기 없음** — 헤더/푸터 어디에도 `last_ym` 미노출(`Company.last_ym`는 있으나 미사용). 원본도 회사명 옆 기준월은 약하나, 추정 신뢰도 위해 푸터 한 줄 권장.
- ⚠️ 출처 카피는 일관(공공데이터 국민연금·추정·참고용)되나, **시계열 차트가 의사난수 생성**(`getMonthlyStats`, `mulberry32`) — "추이"는 실측이 아니라 현재값에서 역산한 합성 시계열. "그때 vs 지금"을 이 위에 얹으면 **허구 서사** 위험. 고지 문구에 "추이는 추정 시뮬레이션" 명시 필요.
- ✅ 위험등급 카피: 탐색(좋소확정/보통/희귀중소)과 상세 라벨 톤 일치.

## 다음 우선순위

### P0 — 신뢰·저비용 마감 (즉시)
1. **회사 상세 기준월 노출** — 푸터에 `데이터 기준 {c.last_ym}` 한 줄. 추정 신뢰의 최소 요건.
2. **시계열 합성 고지** — 차트 캡션에 "추이는 현재 지표 기반 추정 시뮬레이션" 명시. "그때 vs 지금" 도입 전 필수(허구 서사 방지).

### P1 — 원본 시그니처 채우기 (jotso다움 = 바이럴)
3. **추천 2분화 + 배수** — `nearbyCompanies`를 2모드로: 기존 연봉순 + 저위험순. "더 받는 곳" 카드에 `salary/자사salary` 배수("7.9배") 표기. 데이터 접근 동일, 정렬키·배수만 추가 — 최저비용·고효과.
4. **위험도 사다리(더좋소/덜좋소)** — `_topRisk` 또는 점수 정렬 인덱스에서 자기 ±3 슬라이스. 인메모리로 즉시 가능.
5. **공유 버튼(복사 우선)** — 상세 헤더에 `navigator.clipboard` "링크 복사" 클라 컴포넌트. OG 카드는 이미 작동하므로 액션만 붙이면 확산 루프 완성. 카카오는 JS 키 확보 후.
6. **"그때 vs 지금" 카드** — 단, P0-2(합성 고지) 선행 필수. 연도 토글 → 시작값↔현재값 증감 서사. 데이터 진실성 정리 후 착수.

### P2 — 초과 차별화 & 성능
7. **사전 인덱스** — `regionRank`/`salaryPercentile`/`nearbyCompanies`/`searchCompanies`가 모두 매 요청 552k 선형 스캔. 상세 1회 렌더에 전수 스캔이 3~4회 누적. 시군구·업종·점수 버킷 사전 인덱스(모듈 로드 1회)로 정리 — P1 기능 추가의 선행 조건.
8. **연봉 백분위/지역순위 캐싱** — 7의 일부로 통합.
9. **비교(`/compare?a=&b=`)** — 이직 의사결정 도구. 원본에 없는 상위 기능.

> 권장 순서: P0-1·2(신뢰) → P1-3(추천2분화·최저비용) → P1-5(공유버튼) → P2-7(인덱스) → P1-4·6. 추천2분화·공유버튼은 **데이터 함수가 이미 있어** 며칠 내 원본 1:1 도달 가능. "그때vs지금"은 데이터 진실성 정리가 선행이라 의도적으로 뒤로.
