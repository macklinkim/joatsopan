## 종합 — 기능은 원본 1:1 도달, 남은 건 '정리(편집)' 문제

round3-10이 P1로 지목한 회사 상세 4종(위험도 사다리·갈아탈 곳·주변추천·공유 버튼)이 **모두 구현·렌더 확인**됐다(`page.tsx:48~269`, `ShareButton`). 즉 이번 라운드의 결함은 "없는 기능"이 아니라 **중복·일관성·빈 편집**에 있다. 가장 큰 단일 문제는 **면책 고지의 3중 노출**과 **추천 블록(사다리·갈아탈·주변)의 의미 중복**이다.

## (1) 카피 / 톤 일관성 — 추정·단위·라벨

- ⚠️ **면책 고지가 화면당 2~3회 반복**(근거: grep 12곳). 전역 `Footer.tsx:8`("…추정치이며 참고용…특정 기업 비방 목적이 아닙니다")가 **모든 페이지 하단에 이미** 깔리는데, 그 위에 페이지 인라인 고지가 또 있다.
  - 회사 상세: 헤더 마이크로카피(`page.tsx:103`) + 인라인 푸터(`:272`) + 전역 푸터 = **동일 취지 3회**. 모바일 풀페이지 캡처에서 페이지 끝 고지와 전역 푸터 고지가 연달아 보임.
  - `/monthly`·`/awards`(`:21`/`:40`)는 인라인 고지가 전역 푸터와 **문장까지 거의 동일**.
- ⚠️ **"추정 곡선" 톤은 정리됨**(round3-10 P0 반영). 상세 차트 캡션 "월별 추이는 최근월 실측 기반 추정 곡선"(`page.tsx:180`)·`/about` "추정 곡선"(`:49`)이 일치 → 허구 서사 위험은 해소. 좋음.
- ⚠️ **위험 등급 라벨 톤 불일치.** `/explore`는 "좋소 확정(50+)/보통/희귀 중소(<20)"로 **점수 구간을 노출**(`explore:9~13`)하는데, 상세 헤더 배지·게이지는 등급 라벨만 표기. 구간(0~19/20~49/50+)이 `/about:43`에만 있고 상세엔 없어 사용자가 "위험도 0 = 희귀 중소"의 의미를 상세에서 못 잡는다.
- 🟢 단위·숫자: `won`/`num`·`tnum` 일관. "명/만원/%/배/곳/위" 통일.
- ⚠️ **"좋소" 용어 톤 흔들림.** 코너는 "이달의 좋소/별이 된 좋소"로 좋소=대상 전체를 지칭하나, `/explore`·`/about`의 "좋소 확정(50+)"은 좋소=고위험만 지칭. 같은 단어가 두 의미.

## (2) 회사 상세 — 정보 과밀 / 순서 / 추천 블록 중복

- ⚠️ **하단 3블록 의미 중복.** ① 위험도 사다리(`:209`, 같은 시군구 위험도 ±이웃) ② 갈아탈 만한 곳(`:250`, 더 주고 덜 위험) ③ 주변 회사 추천(`:245`, 연봉순). ①과 ③은 **같은 시군구/동 풀에서 정렬키만 다른 리스트**라 사용자에겐 "비슷한 표 3개"로 읽힌다. 모바일 캡처상 사다리→갈아탈→주변이 연속 4스크롤.
- ⚠️ **상단도 카드 4종 연속.** 게이지 헤더 → 지역순위+연봉백분위(`:113`) → 핵심지표4 → 차트3. 정보는 정확하나 "요약 한 줄"이 없어 첫 화면에서 결론(이 회사 위험한가?)을 게이지 외엔 못 잡음.
- 🟢 순서 자체는 합리적(헤더→맥락순위→지표→추이→대안). 문제는 양(편집).
- 권장: ①사다리는 **차트 아래 접이식**으로 강등, ②"갈아탈 곳"을 주변추천 안의 강조 칩으로 흡수(현재도 `NearbyList`가 `mult`(배수) 칩을 이미 보유 — `NearbyList.tsx:29`), ③별도 "갈아탈" 박스 제거 시 중복 해소.

## (3) 빈 상태 — 무결과 처리

- 🟢 **검색 무결과**: "검색 결과가 없습니다." 드롭다운(`SearchBox.tsx:117`). 정상.
- 🟢 **사다리 없음**: `riskLadder`가 시군구 4곳 미만이면 `null` → 섹션 자체 비노출(`page.tsx:209` 조건). 한쪽만 비면 "이 지역 최고/최저 위험도입니다" 안내(`:224`/`:237`). 좋음.
- 🟢 **주변 없음**: `NearbyList` "주변(같은 동) 추천 회사가 없습니다."(`NearbyList.tsx:7`). 단, 실제 scope가 동/시군구/업종/전국 폴백이라 **카피의 "같은 동"이 실제 scope와 어긋날 수 있음**(`page.tsx:61` nearbyDesc는 정확한데 빈 상태 문구만 "동" 고정) → ⚠️ 사소 불일치.
- 🟢 **갈아탈 없음**: `better.length>0`일 때만 박스(`:250`) → 없으면 깔끔히 생략.
- 🟢 **코너 리스트 빈 상태**: `CompanyRankList` "표시할 회사가 없습니다."(`:14`).

## (5) 전역 푸터 ↔ 페이지 인라인 고지 중복 — **이번 라운드 1순위**

- ⚠️ `app/layout.tsx`가 전역 `Footer`를 모든 페이지에 렌더(`:42~44`). 그런데 `monthly:21`·`awards:40`·`explore:77`·`memorial:22`·`company:272`가 **각자 인라인 고지를 또** 둠. 결과: 한 화면에 동일 면책이 **2회**(코너) ~ **3회**(상세).
- 권장: 인라인 고지 5곳 **삭제**, 전역 Footer 한 곳으로 단일화. 페이지별 특수 고지(예: memorial "휴·폐업 여부와 다를 수 있다", 상세 차트 "추정 곡선")만 맥락에 남기고 일반 면책은 전역에 위임.

## 우선순위

### P0 — 편집(저비용·고효과)
1. **면책 고지 단일화** — 인라인 고지 5곳 제거, 전역 Footer로 위임. 화면당 3회→1회.
2. **추천 블록 통합** — 별도 "갈아탈 만한 곳" 박스를 `NearbyList` 배수 칩으로 흡수, 사다리는 접이식 강등. 하단 3블록→1~2블록.

### P1 — 일관성
3. **등급 구간을 상세에도** — 게이지/배지 근처에 "0~19 희귀·20~49 보통·50+ 좋소" 툴팁/캡션 1줄. explore와 동일 기준 노출.
4. **빈 상태 카피 ↔ scope 일치** — NearbyList 빈 문구를 실제 scope(`nearbyDesc`)에 맞춰 동적화.
5. **기준월 표기 일관** — monthly/memorial/awards 서브카피에 `기준월 {DATA_YM}` 추가(home·about·상세와 정렬).

### P2 — 마감
6. "좋소" 용어 이중 의미 정리(고위험=좋소 vs 대상=좋소) — 코너/등급 라벨 톤 통일.
7. 첫 화면 요약 한 줄(헤더에 "한 줄 결론") + NavBar 가로스크롤 단서.

> 핵심: 이번 라운드 결함은 기능 부재가 아니라 **편집 부재**. 면책 3중·추천 3블록만 정리하면 완성도가 한 단계 오른다. 데이터·함수는 이미 다 있어 삭제·재배치 위주의 저비용 작업.
