# Round2-07 — 보안 · 배포 · 법적/윤리 리스크 (실데이터 규모 격상)

점검일: 2026-06-20 / 대상: jotsopan (Next.js 15.5.x, Vercel `jotsopan.vercel.app` 공개 200, GitHub `macklinkim/joatsopan`)

## 점검 범위

실데이터가 8개 시드+500 가공(pass-08 시점)에서 **전국 실명 552,878곳 전수**(`data/companies.json`)로 격상됨에 따라 법적·운영 리스크가 질적으로 달라졌다. 본 회차는 그 격상분을 중심으로 재평가한다.

1. 명예훼손/모욕 리스크 — 실명 55만 곳에 단정 코멘트 + 추정연봉/회전율 공개, 고지문 적정성, 톤 완화안
2. 44MB JSON 커밋/번들 — repo 비대, 서버리스 함수 크기/메모리, 재배포 비용
3. `/api/search`·`/api/nearby` 입력검증·레이트리밋(실데이터 스캔비용 DoS)
4. 비밀/.env/.vercelignore 적정성
5. 보안 헤더(`next.config` 비어 있음)
6. 공공데이터 출처표기·라이선스 고지

근거: `lib/data.ts`, `app/api/search/route.ts`, `app/api/nearby/route.ts`, `app/company/[id]/page.tsx`, `app/layout.tsx`, `app/page.tsx`, `next.config.mjs`, `.vercelignore`, `.gitignore`, `scripts/etl.mjs`, `git ls-files`/`git log`, `data/companies.json` 직접 파싱.

## 발견 (심각도 · 근거)

### F1. 실명 552k + 단정 코멘트 — 명예훼손/모욕 리스크 [치명 / Critical — 최우선]

데이터셋을 직접 파싱한 결과 `cols.name`은 `삼성전자(주)`·`현대자동차(주)`·`쿠팡풀필먼트서비스 유한회사`·`에스케이하이닉스 주식회사`·`엘지전자(주)` 등 **실명 552,878건**이며 `bizNo`(`124810`, `101810` 등)는 실 사업자번호 앞 6자리다(`scripts/etl.mjs`: 국민연금 가입 사업장 내역 CSV 593k행 가공). pass-08의 "실명 의심 8개"가 아니라 **전수가 식별 가능한 실재 법인**이다.

여기에 점수 기반 단정 코멘트가 자동 생성·노출된다(`lib/data.ts:32-39`, `commentForScore`):
- `score>=70` → `"도망치세요. 신호가 강하게 옵니다."`
- `score>=50` → `"꽤 위험합니다. 신중히 보세요."`
- closed → `"휴·폐업 신호가 보입니다. 입사 전 꼭 확인하세요."`

이 코멘트는 회사 상세 상단에 큰 글씨·색상으로 인용 노출되고(`app/company/[id]/page.tsx:71-73`, `text-lg` + `riskColor`), 추정 평균연봉(`:90-96`)·회전율(`:97-103`)·휴폐업 여부(`:104-110`)가 함께 단정적으로 표기된다. 시상식 페이지는 한발 더 나아가 실명 회사를 `🚨 올해의 좋소`·`🥶 박봉 대상`·`🌀 회전문 대상`으로 호명한다(`lib/data.ts:211-217`).

법적 평가: 추정연봉·회전율은 **사실의 적시**이고, 그 산정 근거(`riskScore`, `scripts/etl.mjs:18-25`)는 국민연금 고지금액에서 역산한 *추정*이라 실제와 다를 수 있다. 실명 법인에 대해 사실로 오인될 수치를 적시하고 `"도망치세요"`·`"좋소 확정"` 같은 경멸적 표현을 결합하면 정보통신망법 제70조(사이버 명예훼손)·형법 제307·311조(명예훼손·모욕) 소지가 있다. `"참고용·추정"` 고지가 있어도 위법성 조각(제70조 단서: 비방 목적 부재 + 공익) 요건을 자동 충족하지 않으며, 특히 영리 광고가 아닌 단정·조롱 톤은 비방 목적으로 해석될 여지가 크다. 대규모 실명이라 1건만 분쟁화돼도 정정·삭제·손해배상 청구가 가능하고, 집단적 정정 요청 시 운영 부담이 급증한다.

### F2. 고지문 위치·강도 부족 [높음 / High]

고지문은 `app/page.tsx:26`, `app/company/[id]/page.tsx:149-150`, `app/awards/page.tsx:40`, `app/monthly/page.tsx:21`, `app/memorial/page.tsx:22`, `app/layout.tsx:24`(메타)에 존재한다. 그러나 상세 페이지 고지문은 **페이지 최하단 푸터**에 `text-xs text-outline`(가장 작은 글씨·흐린 색)으로, 상단의 큰 단정 코멘트 대비 시각적 비중이 현저히 낮다(`app/company/[id]/page.tsx:71-73` vs `:149-150`). 또한 (a) 데이터 출처·기준월·산정식 설명 페이지, (b) **정정/삭제 요청 창구**, (c) 면책 약관이 전무하다. 실명 552k 규모에서 정정 창구 부재는 분쟁 시 즉시 약점이 된다.

### F3. 44MB JSON 커밋/번들 [높음 / High]

`git ls-files`에 `data/companies.json`이 추적되고 git blob 크기 **45,160,821바이트(약 43MB)**가 HEAD에 커밋돼 있다(`git cat-file -s`). 문제:
- **repo 비대**: 데이터 갱신마다 신규 43MB blob이 히스토리에 누적 → clone/fetch·GitHub 저장 비용 증가, 50MB 경고/100MB 하드리밋 근접.
- **서버리스 함수 크기**: `next.config.mjs`의 `outputFileTracingIncludes`가 `"/**"`에 `./data/companies.json`을 포함시켜 **모든 함수 번들에 43MB가 트레이싱**된다. Vercel 람다 비압축 250MB 한도에 큰 폭으로 잠식하고, 콜드스타트 시 `fs.readFileSync` + `JSON.parse`(`lib/data.ts:18-20`)로 43MB를 동기 파싱해 메모리·콜드스타트 지연이 크다.
- **주석 불일치**: `next.config.mjs:3` 주석은 "8MB"라 적혀 있으나 실제 43MB — 갱신 누락.
- 참고로 `.vercelignore`는 `data/raw`(115MB 원본 CSV)·`*.png`·`docs/review`만 제외하므로 가공본 43MB는 배포에 포함된다(의도된 동작이나 위 비용 수반).

### F4. API 입력검증·레이트리밋 부재 — 실데이터로 위험 격상 [높음 / High]

`app/api/search/route.ts:6`은 `q`를 길이·문자 제한 없이 받고, `searchCompanies`(`lib/data.ts:132-146`)는 매 요청 **552,878건 전수 선형 스캔**(`nameLc[i].includes` + `bizNo.includes`)을 수행한다. pass-08 시점(508건 스캔)과 달리 이제 단일 검색이 55만 문자열 포함검사라 **요청당 비용이 1000배**다. 레이트리밋이 없어(`route.ts`에 미들웨어/제한 없음) 짧은/광범위 쿼리를 고빈도로 던지면 함수 CPU·메모리를 고갈시키는 저비용 DoS·스크래핑 표면이 된다. `nearbyCompanies`(`lib/data.ts:151-176`)도 동일 — `id`당 최대 4티어 × 55만 순회(`collect`)라 단건 호출 비용이 크고 `id` 형식 검증이 없다. 인젝션 경로는 없으나(메모리 배열·`Map.get`) 비용·가용성 리스크가 핵심이다.

### F5. 보안 헤더 전무 [중 / Medium]

`next.config.mjs`에 `headers()`가 없어 `Content-Security-Policy`·`X-Frame-Options`·`X-Content-Type-Options`·`Referrer-Policy`·HSTS 미설정. 실명 위험도 화면을 제3자가 iframe으로 임베드(클릭재킹)해 "이 회사는 좋소" 화면을 유포하는 행위를 막지 못한다 — 명예훼손 리스크(F1)와 결합 시 피해 확산 경로가 된다.

### F6. 공공데이터 출처표기·라이선스 고지 미흡 [중 / Medium]

`scripts/etl.mjs`는 국민연금 '가입 사업장 내역' CSV를 원천으로 명시하나, 앱 표면의 출처 표기는 `"공공데이터(국민연금)"` 문구뿐이다(`app/page.tsx` 등). 공공데이터포털 제공 데이터는 통상 출처표시(예: 이용허락범위·기관명·갱신주기) 고지를 요구한다. 정확한 데이터셋명·제공기관·기준월(`DATA_YM`=2026-04)·이용약관 링크가 별도 명시돼 있지 않아 라이선스 준수가 불완전하다. 갱신주기/재배포 조건 위반 가능성도 검토 필요.

### F7. 비밀/.env/.vercelignore — 양호 [정보 / Info]

- 디스크에 `.env*` 없음, 코드에 하드코딩 토큰/키 없음(외부 API 미호출, 데이터는 로컬 JSON).
- `.gitignore`가 `.env`·`.env*.local`·`.vercel`·`/node_modules`·`/.next`·`*.pem`·`/data/raw/`를 모두 제외(`.gitignore`). 단 `.vercel`이 두 번 기재(중복·무해).
- `.vercelignore`가 `data/raw`(115MB 원본)·`*.png`·`.playwright-mcp`·`/tmp`·`docs/review` 제외 — 원본 CSV가 배포에 안 들어가는 점은 적정.
- 결론: 비밀 노출 위험 없음. 단 F3대로 가공본 43MB 커밋은 별개 문제다.

## 구체적 개선안

### 즉시 (법적 리스크 차단 — 최우선, F1·F2)

1. **코멘트 톤을 단정 → 추정·정보형으로 전면 완화** (`lib/data.ts:32-39`). 예:
   - `"도망치세요. 신호가 강하게 옵니다."` → `"위험 지표가 높게 추정됩니다. 직접 확인을 권합니다."`
   - `"꽤 위험합니다. 신중히 보세요."` → `"평균 대비 주의 지표가 보입니다(추정)."`
   - closed → `"가입자 추이상 휴·폐업 가능성이 추정됩니다. 실제와 다를 수 있습니다."`
   - 모든 코멘트에 `(추정)` 명시, 1·2인칭 명령형("도망치세요")·경멸 표현("좋소 확정") 제거.
2. 시상식 카테고리명 완화 (`lib/data.ts:211-217`) — `🚨 올해의 좋소`·`🥶 박봉 대상` 등 조롱형 라벨은 실명과 결합 시 모욕 소지. "추정 지표 상위" 식 중립 라벨로, 또는 시상식은 가명/익명 표시로.
3. 상세 페이지 고지문을 **코멘트 인접 상단으로 이동·확대**(`app/company/[id]/page.tsx:71-73` 근처), 작은 푸터(`:149-150`)에만 두지 말 것. + **정정/삭제 요청 창구**(이메일/폼)와 산정식 설명 링크 추가.
4. 운영 정책 검토: 실명 552k 전수 공개의 정당성(공익·비영리·출처) 문서화, 정정 요청 처리 절차 마련. 개인용/포트폴리오라면 실명 비노출(검색 결과만 마스킹 또는 동의 기반)도 강하게 권장.

### 데이터/배포 (F3)

5. 43MB JSON을 git에서 분리: Vercel Blob/외부 스토리지 또는 빌드시 다운로드로 전환하고 `.gitignore`에 추가, 기존 blob은 히스토리 정리(필요 시) 검토. 최소한 `outputFileTracingIncludes`를 데이터를 쓰는 라우트로만 한정해 전 함수 번들 포함을 막을 것.
6. `next.config.mjs:3` 주석 "8MB" → 실제 크기로 수정. 콜드스타트 부담이 크면 런타임 부분 로드(인덱스 분할)·DB 전환 검토.

### 코드 (F4·F5)

7. 입력 검증: `q`는 `trim().slice(0, 50)` + 길이<2면 빈 배열 즉시 반환(`app/api/search/route.ts`). `id`는 영숫자 형식 정규식 검증 후 조회(`app/api/nearby/route.ts`).
8. 레이트리밋: `@upstash/ratelimit`+Upstash 또는 미들웨어 IP 기반 제한. 55만 전수 스캔 비용 때문에 우선순위 상향. 검색은 사전 구축 인덱스(접두어 트라이/prebuilt) 도입으로 요청당 비용 자체를 낮추는 것도 병행.
9. 보안 헤더: `next.config.mjs`에 `async headers()` — `X-Content-Type-Options: nosniff`, `X-Frame-Options: DENY`(또는 CSP `frame-ancestors 'none'`), `Referrer-Policy: strict-origin-when-cross-origin`, HSTS.

### 고지/라이선스 (F6)

10. 출처 페이지 신설: 데이터셋 정식명·제공기관·기준월·이용허락범위·갱신일·재배포 조건을 명시하고 약관/고지에서 링크. 공공데이터 이용약관의 출처표시 의무 준수.

## 반성점

- pass-08은 "실명 의심 8개"로 봤으나, 이번엔 데이터셋을 직접 파싱해 **전수 552,878건이 실명·실 사업자번호**임을 확정했다(추정 아님). 데이터 성격 변화로 F1을 High→**Critical**로, F4를 실스캔비용 1000배 근거로 격상한 것이 본 회차 핵심이다.
- 코멘트가 `commentForScore`로 일반화돼 8개 하드코딩보다 *덜* 위험해 보일 수 있으나, 실은 **자동으로 55만 곳 전부에 단정 코멘트가 붙는** 구조라 위험 표면은 반대로 폭증했다 — 코드 구조만 보고 "개선됐다"고 오판하지 않도록 데이터 규모와 결합해 평가했다.
- 실 사업자번호 앞 6자리 진위·국민연금 데이터의 정확한 이용약관 조항은 외부 확인 없이 단정 못 한다. "실명·실 사업자번호로 보이며 출처표시 의무 검토 필요"까지만 근거 있게 서술하고 그 이상은 권고로 표시했다.
