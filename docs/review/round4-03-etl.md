# Round4-03 · ETL / 실데이터 정확성 재검증 (전체행 dedup·시도통일 이후)

대상: `scripts/etl.mjs`, `lib/scoreCore.mjs`(SSOT), `data/companies.json`, `data/raw/nps.csv`
데이터: 국민연금 가입 사업장 내역 2026-04. 원본 593,075행(22필드, 빈줄 1) → 완전중복 1행 제거 → 활성 적재 552,878곳 / 휴폐업 200곳.
검증방법: `companies.json` 전수 통계 + 원본 CSV(EUC-KR) 재집계로 교차검증. Round3 핵심 지적(dedup 키가 별개 사업장 185건 과다제거, 시도명 레거시/신규 혼재)의 수정 반영 여부 재확인이 핵심.

## 점검범위
- 시도명 통일(`SIDO_ALIAS`) 후 잔존 레거시명, 시군구/동 매칭 무결
- 전체행 dedup이 실제 중복만 제거하는지(과다/과소)
- 주소파싱 결측률·통합시·세종
- 점수 분포·estSalary 상한 압축·업종 중앙값 폴백
- 인터닝 인덱스 범위 무결, score 컬럼 vs companyAt 재계산 일치
- 휴폐업 200곳 적정성

## 발견 (심각도 · 근거)

### [확인 통과] 1. 시도명 통일 — Round3 지적 전면 해소
- `SIDO_ALIAS`(`etl.mjs:95-99`)로 강원도→강원특별자치도, 전라북도→전북특별자치도, 제주도→제주특별자치도 매핑. 전수 측정 결과 **레거시명(강원도/전라북도/제주도) 인터닝 엔트리 0개**.
- sido 리스트 18개 = 17개 광역 + 빈문자열 1개. 모두 현행 명칭(강원특별자치도·전북특별자치도·제주특별자치도·세종특별자치시 포함). Round3에서 분리되던 24건(강원도 6+전라북도 18)이 현행명으로 흡수됨.
- 시군구/동 역참조 매칭 무결: sigungu 235개 중 동/읍/면으로 끝나는 오염값 0, 광역시 자치구 48개·통합시 패턴 정상.

### [확인 통과] 2. 전체행 dedup — Round3 과다제거 완전 교정, 실제 중복만 제거
- `etl.mjs:35-37`이 **완전중복(라인 전체 22필드 동일)** 만 제거하도록 변경 확인. 원본 재집계 결과 **제거 1행**(593,075 파싱 → 552,878 활성, 코드와 정확히 일치).
- Round3의 `name+bizNo+bdong` 키(191건 제거, 그중 185건이 별개 사업장)가 폐기됨. 검증: 같은 name+bizNo+bdong 키 충돌은 활성에 **176건 잔존하지만 의도적으로 모두 보존**됨. 표본 대조로 별개 사업장임을 확인:
  - `아그네스수산(주)|201814|도렴동`: 15명 vs 30명(notice 7.17억 vs 12.29억) — 별개 분소, 둘 다 보존.
  - `국민연금공단|219820|만성동`: 5,878명 vs 1,174명 — 둘 다 보존.
  - `한화호텔앤드리조트(주)|101813`: 본사 265명 / 포천 20명 / 해운대 96명 / 보령 24명 등 15개 분소 전부 보존.
- bizNo 6자리 마스킹으로 유니크 식별 불가(prefix 재사용)인 상황에서 **완전행 일치 기준이 정답**: 과다제거(별개 사업장 삭제) 0, 과소제거 무관(완전동일 1건만 존재). Round3 권고안 그대로 반영됨.

### [확인 통과] 3. 주소파싱 — 통합시·세종·결측 모두 정상
- `parseRegion`(`etl.mjs:101-108`) 전수 측정:
  - **세종 3,222행 전부 `[세종, "", 동]`**(`etl.mjs:104`) — sigungu 결측이 곧 정상.
  - 통합시(성남시 분당구 등) `sigungu="시 구"` 병합 정상, dong 슬롯에 '구' 오염 없음.
- 결측률: sido 109(0.020%), sigungu 3,331(0.602% = 세종 3,222 + 비세종 109), dong 110(0.020%). 비세종 sigungu 결측 109건 = sido 결측과 동수로, 원본 jibun 공백행에 기인(ETL 파서 결함 아님).

### [경미] 4. estSalary 상한소득월액 압축 — 코드 캡 아님, 문서화 완비
- 공식 `(notice/members/0.09)*12/10000`(`scoreCore.mjs:25-28`) 단위·요율 타당, 코드상 상한 캡 없음.
- 전수 분포(만원): min 507 / p50 3,990 / p90 5,766 / p99 7,383 / **max 8,069**. >1억(10,000) 0건, salary==0 적재행 0. 고연봉 구간이 NPS 기준소득월액 상한으로 8,000만원대에 눌리는 **자연 천장**이며, `app/about/page.tsx:48`에 "추정 연봉은 기준소득월액 상한 때문에 고연봉 구간이 과소평가될 수 있다"로 명시됨.
- 영향: 점수 s2(중앙값 대비)는 과소추정만 발생 → 위험도 상방 왜곡 없음. 변별력 손실만 존재, 문서화로 커버됨.

### [경미] 5. 업종 중앙값 폴백 — 소표본 노이즈 잔존
- 업종 1,575개. allMedian(3,990) 폴백 적용 업종 단 2개이나, **median이 allMedian과 동일한 회사 787곳**은 표본 충분 업종이 우연히 3,990인 경우 포함이라 무해.
- 단 N<5 단일/소표본 업종이 곧 중앙값이 되는 노이즈는 잔존(Round3 미반영). 점수 영향 제한적이나 N<5 폴백 권장 유지.

### [확인 통과] 6. 점수 분포 + score 컬럼 vs 재계산 일치
- **score 컬럼 전수 재계산 일치: 활성 552,878/552,878 불일치 0, 휴폐업 200/200 불일치 0**(`scoreCore.riskScore`로 members/salary/turnover/업종median 재산출 = 저장값 완전 일치). ETL의 score 사전계산과 런타임 SSOT가 결정적으로 동일.
- 점수 분포: min 0 / max 82 / avg 19.63. 라벨 희귀중소(<20) 343,028(62.0%) / 보통 207,845(37.6%) / 좋소확정(≥50) 2,005(0.4%). max 82 = s1 16+s2 35+s3 31(5명·저연봉·고turnover) 조합으로 100 상한 내 정상. turnover max 200(코드 캡 아님, 일용직 현장 자연 천장).

### [확인 통과] 7. 인터닝 무결성 — 완전
- 전 인덱스 컬럼 범위이탈 0(active 552,878행 × 4컬럼): sidoIx(<18), sgIx(<235), dongIx(<3902), indIx(<1575). closed 200행도 동일 테이블 참조, 범위 정상.

### [경미] 8. 휴폐업 200곳 — 의도대로 동작, 풀 고정
- 탈퇴(status≠1) 39,554곳 중 members≥1은 245곳, 그중 상위 200곳 적재(`etl.mjs:134-135`). members min 3 / max 1,047.

## 개선안
1. (권장) 업종 표본 N<5 업종은 allMedian/상위 업종군 중앙값 폴백으로 단일표본 노이즈 제거(Round3부터 이월).
2. (권장) estSalary 상한 근접 행에 "상한 이상" 플래그 추가로 과소추정 오해 방지(about 문구는 이미 존재).
3. (선택) closed 풀 200 고정값 상수 분리.
4. (선택) sido코드와 jibun 불일치(비세종 sigungu 결측 109건 등) 원본 이상행 로깅.

## 반성점
- Round3 두 핵심 지적이 모두 반영 확인됨: (1) dedup이 완전행 일치로 바뀌어 별개 사업장 185건이 복원(552,693→552,878), 검증은 "Round3가 제거하던 176건 충돌이 이번엔 전부 보존되는가"를 표본 대조로 재측정한 것이 결정타. (2) 시도명 통일로 레거시 엔트리 0 확인.
- 새 확인 포인트는 score 컬럼 전수 재계산 일치(552,878 + 200 = 0 불일치)로, ETL 사전계산과 SSOT 런타임 재계산이 일관됨을 수치로 못박음.
- "estSalary max 8,069"·"turnover max 200" 같은 딱 떨어지는 상한은 이번에도 코드 캡 부재를 재확인하고 데이터 자연 천장(기준소득월액 상한·일용직 현장)으로 귀속시킴. about 페이지 문서화 존재까지 교차확인.
