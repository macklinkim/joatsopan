## 발견 (심각도 / 근거)

### F-1. [높음] 옐로우(#FEE500) 텍스트의 명도 대비 실패 — 가장 흔한 "보통(20~50)" 밴드 직격
- 근거: `lib/format.ts` L20-24 `riskColor` 는 score 20~49 에 **`#FEE500`(카카오 옐로우)** 를 반환한다. 이 값이 **흰 배경 위 텍스트 색**으로 그대로 쓰인다 — 상세 헤더 점수/한줄평(`company/[id]/page.tsx` L71 `color: riskColor(...)`로 comment, RiskGauge L49 게이지 숫자), 시상식 배지 텍스트(`awards/page.tsx` L27), MetricCard/배지 전반.
- 실증(`rd2-midrisk-yellow.png`, score 26 회사): 게이지 숫자 "26" 과 한줄평 "나쁘지 않아 보입니다." 가 **흰 바탕에 옐로우**로 거의 안 읽힌다. `#FEE500` on white 대비비 ≈ **1.3:1** (WCAG AA 4.5:1 대폭 미달). "보통" 배지는 옐로우 10% 배경(`riskColor()1a`)에 옐로우 글자라 더 나쁘다.
- 영향: score 20~49 은 실데이터에서 매우 흔한 구간(위험도 분포 0~82, 중앙대역). 즉 **다수 회사 상세 페이지에서 점수·한줄평이 사실상 판독 불가**. 게이지 트랙(stroke)으로서의 옐로우는 괜찮지만(`RiskGauge` L36), 텍스트 색으로 재사용한 게 문제. 별표(★) 1순위 결함.

### F-2. [높음] 동명 회사 구분 단서 부족 — 검색 드롭다운에 지역(시군구·동)이 없음
- 근거: `SearchBox.tsx` L87-89 드롭다운 행은 **상호명 + 업종 + 직원수(명)** 만 표시한다. `app/api/search/route.ts` 의 `SearchResult` 에 `sigungu`/`dong` 자체가 매핑되지 않는다(types.ts L46-52 에도 지역 필드 없음).
- 실증(`rd2-search-dup.png`, q="준오헤어"): "준오헤어 · 두발 미용업 · 21명", "준오헤어 · 두발 미용업 · 19명", "준오헤어 · 두발 미용업 · 18명" … 이 **상호·업종 동일, 직원수만 다른 행**으로 줄줄이 나온다. 준오헤어는 37개, 유디치과의원/블리비의원은 27개 존재 — 프랜차이즈/체인은 업종까지 같아서 사용자가 "우리 동네 그 지점"을 고를 단서가 직원수 숫자뿐이다.
- 대비: 상세/순위/주변 화면은 지역을 보여준다(`CompanyRankList` L30-32 `{sigungu} {dong} · {industry}`, 상세 헤더 L69). 정작 **진입점인 검색만 지역이 빠져** 가장 구분이 필요한 곳에서 단서가 약하다.
- 영향: 동명사가 흔한 한국 실데이터에서 잘못된 지점을 클릭할 확률이 높다. F계열 중 사용성 타격이 큰 항목.

### F-3. [중간] 매우 긴 실제 회사명 — 오버플로는 없으나 레이아웃 지배/잘림 트레이드오프
- 상세 헤더(`company/[id]/page.tsx` L65-67 `<h1>`): `truncate` 없이 자연 줄바꿈. 실증(`rd2-longname-mobile.png`, 87자명)에서 **헤더가 4줄로 늘어** 게이지/지표를 한참 밀어낸다. 가로 오버플로·잘림은 없음(양호) 이나 79~87자 현장명-사명에선 헤더가 화면을 지배한다. CJK+ASCII 혼합 장문이라 `break-words`/`min-w-0` 보강 시 더 안전.
- 검색 드롭다운(`SearchBox.tsx` L87): 상호명 `shrink-0` + 업종 `truncate flex-1`. 즉 **상호명은 줄지 않아** 87자 상호가 오면 업종/직원수를 밀어내고 한 줄을 가득 채울 수 있다(업종만 truncate). 장문 상호에서 행이 깨질 여지.
- 시상식 카드(`awards/page.tsx` L32 `truncate`): 양호 — `rd2-awards.png` 에서 "주식회사창원기전-(일용)(이천)26년M16 PH1 DRAM 투자 H-up…" 로 깔끔히 말줄임.
- NearbyList/CompanyRankList 상호(`NearbyList.tsx` L24, `CompanyRankList.tsx` L29): 둘 다 `truncate` + `min-w-0 flex-1` 로 양호. NearbyList 는 부가정보(업종·지역)가 아예 없어 장문에도 안전하지만, 그만큼 주변 추천에서 회사 구분 단서가 직원수/연봉뿐(F-2 와 동류의 약점).

### F-4. [중간] NavBar 모바일 줄바꿈 — 4개 메뉴가 좁은 폭에서 2줄로 깨짐
- 근거: `NavBar.tsx` L21-22 메뉴 `<ul>` 는 `flex` 만 있고 가로 스크롤/햄버거 없음. 링크 라벨이 "이달의 좋소"/"별이 된 좋소"/"좋소 시상식"/"좋소 게임" 으로 길다.
- 실증(`rd2-search-dup.png`, 390px): 로고 + 4링크가 한 줄에 안 들어가 각 라벨이 **"이달의 / 좋소", "좋소 시 / 상식", "좋소 게 / 임"** 처럼 2줄로 쪼개져 헤더가 어수선하다. 오버플로(가로 스크롤바)는 없지만 미관·탭 타깃이 나빠진다.
- 영향: 모든 페이지 상단에 상시 노출되는 글로벌 요소라 체감도 높음. 햄버거 메뉴 또는 가로 스크롤 처리 권장.

### F-5. [낮음~중간] 회전율 클램프(200%)·0% 다수가 차트/순위/시상식에서 정보 손실
- 근거: 실데이터 회전율은 [0,200] 으로 **상한 200 클램프**. `topRiskCompanies`(monthly)·`awards`(회전문 대상)는 회전율로 정렬/선정한다.
- 실증(`rd2-awards.png`): "회전문 대상" 수상사 회전율이 **정확히 200%** — 200 에 걸린 회사가 다수 동률이라 "가장 높은 곳" 선정이 사실상 임의(동률 중 첫 인덱스). monthly 의 `c.cur_turnover >= 100` 위험 강조(monthly/page.tsx L17)도 200 클램프 위에서는 변별력이 둔하다.
- 0% 쪽: 318,636건(57.6%)이 회전율 0. 상세의 입·퇴사 차트(`HireLeaveChart`)는 `maxV = Math.max(1, ...)`(L23) 로 0 데이터에서도 0 나눗셈 없이 빈 막대로 안전 렌더 — **차트는 안 깨짐**(양호). 다만 "회전율 0%/안정적" 표기가 절반 이상에 동일하게 붙어 지표 변별력이 낮다(데이터 특성, UI 버그 아님).
- LineChart 극단값: 변동 0(직원수 평탄)일 때 `span===0 → minV=0`(L39) 분기로 평선 처리, `maxData*1.08 || 1`(L40)·`/(maxV-minV||1)`(L42) 가드로 **0 나눗셈/NaN 방지**됨 — 단일점/평탄 시리즈 견고(양호). baseline 이 데이터보다 한참 높으면 `▲` 상단 표기로 전환(L77-84) — 삼성/중위험 캡처 모두 정상 동작 확인.

### F-6. [양호 — 검증됨] 큰 수 포맷·정렬, 게이지/차트 극단값
- 큰 수: `num`/`toLocaleString`(천단위) + Tailwind `tnum`(tabular-nums) 광범위 적용. 실증(`rd2-biggest-metrics.png`): "125,614명", "7,897만원", LineChart Y축 "135,663 / 121,821 / 107,979 / 94,137" 모두 자릿수 정렬 깔끔. 게이지 score 0(삼성)·26(중위험) 모두 정상 렌더, 포인터 좌우 끝 클램프(`RiskGauge` L25 `Math.max(0,Math.min(100,score))`).
- 연봉 `won`: 활성 최대 8,069만원이라 억 분기 미발현이나 로직 자체는 정상(`format.ts` L1-9). 단 generated 히어로/미래 데이터에 1억+ 들어오면 분기 동작.
- 게임 단서(`GuessGame.tsx` L89-91): 직원수/연봉/회전율을 `won`/`tnum` 으로 표기, 상호 `truncate`(L82) — 장문·큰수 모두 안전.

### F-7. [정보] memorial 빈 상태는 현재 미발현 (휴폐업 200건 존재)
- 근거: `data/companies.json` 의 `closed` 셋에 **200건** 존재 → `closedCompanies(40)`(data.ts L190-194) 가 비지 않는다. 따라서 `CompanyRankList` 의 빈 상태("표시할 회사가 없습니다." L15)와 `MemorialPage` 는 정상 목록을 렌더하고 **0건 빈 화면은 지금 데이터에선 발생하지 않음**.
- 다만 빈 상태 카피는 일반형("표시할 회사가 없습니다.")이라, 만약 휴폐업 0건 데이터로 교체되면 memorial 의 추모 맥락("고인의 명복을 빕니다")과 동떨어진 무미건조한 문구가 노출된다. 페이지 전용 빈 상태 메시지 부재(낮은 우선순위).

## 개선 제안 (우선순위)

- **P-1 [F-1, 높음]** 텍스트용 옐로우를 분리. 게이지 트랙엔 `#FEE500` 유지하되, **텍스트/배지 글자색은 진한 앰버(예: `#9A6B00`/`#8A5A00`)** 로 교체해 흰 배경 대비 4.5:1 확보. `riskColor` 와 별도로 `riskTextColor` 헬퍼 추가 권장.
- **P-2 [F-2, 높음]** 검색 결과에 **지역(시군구·동) 추가**: `SearchResult`/`route.ts`/`SearchBox` 행에 `{sigungu} {dong}` 노출(업종 옆 또는 2번째 줄). 동명 회사 식별의 핵심 단서.
- **P-3 [F-3]** 상세 `<h1>` 에 `break-words` (+필요시 `line-clamp-3` 와 전체명 title 속성), SearchBox 상호명 셀에 `truncate min-w-0` 적용해 장문 상호의 행 깨짐 방지.
- **P-4 [F-4]** NavBar 모바일: 햄버거 토글 또는 `overflow-x-auto whitespace-nowrap` 로 가로 스크롤화하여 라벨 줄바꿈 제거.
- **P-5 [F-5]** 회전율 클램프(200%) 표기를 "200%+" 로 명시하고, awards/monthly 동률 타이브레이커(예: 직원수·연봉) 추가로 "회전문 대상" 선정 임의성 완화.
- **P-6 [F-7]** memorial 전용 빈 상태 카피 추가(0건일 때 추모 톤 유지).

## 종합

차트·게이지의 **수치 견고성은 양호**하다 — 125,614명/7,897만원 같은 큰 수 포맷·`tnum` 정렬 정상, 회전율 0%·평탄 시리즈·baseline 초과 등 극단값에서 0 나눗셈/NaN 없이 안전 렌더(가드 확인), 게임/시상식/순위 리스트의 장문 상호는 `truncate` 로 처리됨. 반면 **실데이터 특유의 두 결함이 두드러진다**: (1) "보통(20~50)" 밴드에 쓰이는 옐로우 텍스트의 명도 대비 실패로 흔한 구간 회사의 점수·한줄평이 판독 불가(F-1), (2) 동명 회사(준오헤어×37 등)가 흔한데 검색 드롭다운에 지역 단서가 없어 올바른 지점 선택이 어려움(F-2). 더불어 장문 상호의 상세 헤더 지배(F-3), 모바일 NavBar 줄바꿈(F-4) 이 다듬을 지점. 휴폐업은 200건 존재해 memorial 빈 상태는 현재 미발현(F-7).
