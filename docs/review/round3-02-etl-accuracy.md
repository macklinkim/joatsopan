# Round3-02 · ETL / 실데이터 정확성 재검증 (중복제거·주소파싱 이후)

대상: `scripts/etl.mjs`, `lib/scoreCore.mjs`(SSOT), `data/companies.json`, `data/raw/nps.csv`
데이터: 국민연금 가입 사업장 내역 2026-04. 원본 593,075행 → 유니크 592,884 → 활성 적재 552,693곳 / 휴폐업 200곳.
검증방법: `companies.json` 전수 통계 + 원본 CSV(EUC-KR) 재집계로 교차검증. Round2 지적사항(주소파싱 통합시/세종)의 수정 반영 여부를 재확인하는 것이 핵심.

## 점검범위
- 주소파싱: 통합시(성남시 분당구)·세종·광역시 자치구 정확도, sigungu/dong 결측률
- 중복제거 키(name+bizNo+bdong) 후 잔존 중복 / 과다제거
- estSalary 상한소득월액 미반영(고연봉 압축) 영향 재측정
- 업종 중앙값·점수 분포 이상치
- 휴폐업 200곳 적정성
- 인터닝 무결성(인덱스 역참조 범위)

## 발견 (심각도 · 근거)

### [확인 통과] 1. 주소파싱 — Round2 버그 전면 해소
- `parseRegion`(`etl.mjs:95-102`) 접미사 규칙 재작성 확인. 전수 측정 결과:
  - **dong 슬롯이 '구'로 끝남: 110,014(19.9%) → 8건(0.00%)**. 통합시(경기도 성남시 분당구 …)가 `sigungu="성남시 분당구"`로 정상 병합됨(통합시 패턴 109,976건 = 19.90%).
  - **sigungu 오염(동/읍/면이 시군구 슬롯에): 33개 엔트리 → 0개**. 현재 sigungu 리스트 235개 중 동/읍/면으로 끝나는 값 0.
  - **세종 dong 결측: 3,206 → 0건**. 세종 3,222행 전부 `[세종, "", 동]` 정상(`etl.mjs:98`).
- 결측률 전반 양호: sido 결측 109(0.02%), sigungu 결측 3,331(0.60% = 세종 3,222 + 비세종 109), dong 결측 110(0.02%). 결측 109건은 원본 jibun이 공백("    ")인 행으로 ETL 책임 아님(예: (주)우일씨엔에스, 부천일신초등학교).

### [경미] 2. 잔존 8건 dong='구' / 16건 세종 오분류 — 원본 데이터 오염
- 남은 8건은 모두 sido=세종인데 jibun에 타지역 '구' 주소가 박힌 원본 이상행(예: bizNo 355810 "주식회사리소리우스" jibun=종로구, 431870 jibun=서구). bdong코드는 세종(3611…)이나 jibun 문자열이 어긋남 → 동일 bizNo prefix가 수백 개 사업장에 재사용되며 jibun이 뒤섞인 결과. ETL 파서 결함 아님(원본 sido=36/jibun 불일치). 영향 미미(8건).

### [중] 3. 중복제거 키(name+bizNo+bdong) — 과다제거: 191건 중 185건이 별개 사업장
- 재현: 키 충돌 191건 제거(`etl.mjs:35-37`)는 코드와 정확히 일치(활성 552,693 재현 확인).
- 그러나 **bizNo는 유니크 식별자가 아님**: 552,693행에 유니크 bizNo 32,241개뿐(6자리 마스킹 prefix). 같은 prefix가 전혀 다른 회사·주소 수백 건에 재사용됨(검증: bizNo 266880이 서울 성동구·금천구·부산진구 등 전국 분포).
- 제거된 191건을 첫행과 대조: **완전동일(members+notice+jibun 일치) 6건, 차이있음 185건**. 즉 185건은 같은 이름·같은 법정동·같은 6자리 prefix를 공유하는 **별개 분소(分所)**다. 예:
  - `(재)천주교서울대교구유지재단|201820|중구 명동2가`: A 31명 vs B 1,125명 — 둘 다 실재 사업장인데 B가 삭제됨.
  - `국민연금공단|219820|전주 만성동`: 5,878명 vs 1,174명.
  - `한화호텔앤드리조트|101813|중구`: 265명 vs 30명.
- 영향: 대형 법인의 본사/지사 중 하나가 누락(과소집계). 규모는 작으나(185건/55만) 일부 대형 사업장이 통계에서 사라짐. 키에 jibun(또는 members/notice)을 포함하거나, 원본의 정식 13자리 사업자번호가 없으면 dedup을 보수적으로(완전행 일치) 적용 권장.

### [중] 4. estSalary 상한소득월액 미반영 — 고연봉 압축 재확인
- 공식 `(notice/members/0.09)*12/10000`(`scoreCore.mjs:25-28`) 단위·요율 타당. 그러나 상한소득월액(2025 약 617만원) 캡 존재.
- 전수 분포(만원): min 507 / p50 3,990 / p90 5,766 / p99 7,383 / **max 8,069**. 이론상 1인 캡 ≈ 617만×12 ≈ 7,404. **7,300~7,500 군집 2,376건, >7,400 5,337건** — 캡 도달군 존재 확인. salary==0 적재행 0.
- 영향: 실연봉 7,400만원↑ 기업이 ~7,400만원으로 눌림. 점수 s2(중앙값 대비)는 과소추정만 발생(위험도 상방 왜곡 없음). 고임금 변별력만 상실 — 문서화 필요, 점수 영향은 제한적.

### [경미] 5. 업종 중앙값·점수 분포 — 설계 양호, 소표본 노이즈 잔존
- 업종 1,575개 중앙값: min 1,285 / p50 3,979 / max 7,871. allMedian(3,990) 폴백 업종 단 2개. 상위(재보험 7,871·금융시장관리 7,677), 하위(유사의료 1,285·택시운송 1,569) 모두 도메인 상식과 합치.
- 단, **적재 1건뿐 업종 50개, 5건미만 156개** — 단일표본이 곧 중앙값이 되는 노이즈 잔존(Round2 지적 미반영). N<5 폴백 권장.
- 점수 분포: p50 17 / p99 43 / max 82. 라벨 희귀중소(<20) 62.0% / 보통 37.6% / 좋소확정(≥50) 0.4%(2,002곳). 활성 max 82 = s1 16+s2 35+s3 31 조합(5명·저연봉·turnover 200)로 정상 산출, 100 상한 내. 이상치 없음.
- turnover: max 200(코드 캡 아님 — `scoreCore.mjs:20-23`에 캡 없음). ==200 정확히 474건, >200은 0건. 모두 건설 일용직 현장(이름에 "일용/공사" 포함)으로 데이터 자연 천장. 정상.

### [경미] 6. 휴폐업 200곳 — 의도대로 동작, 풀 고정
- status=1 553,335 / status=2(탈퇴) 39,549 / 기타 0. 탈퇴 중 members≥1만 closedRows → 상위 200건 적재(`etl.mjs:128-129`).
- closed 200곳: members min 3 / p50 5 / max 1,047. score min 46(전부 is_closed → s4=30 가산 확인). 인터닝 테이블은 활성과 공유(범위 정상).
- "별이 된 좋소" 풀이 200 고정값임은 명시 필요(상수 분리 권장). 탈퇴 39,549 중 대부분 members<1로 폐기되는 것은 데이터 공란이라 타당.

### [확인 통과] 7. 인터닝 무결성 — 완전
- 전 인덱스 컬럼 범위이탈 0: sidoIx(0~19/20), sgIx(0~234/235), dongIx(0~3901/3902), indIx(0~1574/1575). closed 컬럼도 동일 테이블 참조, 범위 정상.
- 인터닝 리스트 자체 중복값 0(sido/sigungu/dong/ind code 전부 유니크=리스트길이). 미사용 엔트리 0(낭비 없음). 역참조 일관성 완전.

### [경미] 8. 시도명 레거시/신규 혼재
- `강원도`(6) + `강원특별자치도`(14,933), `전라북도`(18) + `전북특별자치도`(17,751)가 별도 인터닝 엔트리로 공존. 원본 혼재이며 ETL 책임 아니나, 지역 필터/집계 시 양쪽을 합산하지 않으면 24건이 분리됨. 정규화 매핑 권장.

## 개선안
1. (필수) dedup 키에서 비유니크 bizNo 의존 완화: `name+bizNo+bdong+members+notice` 또는 완전행 해시로 변경해 185건 별개 사업장 오삭제 방지. 정식 사업자번호가 없으므로 보수적 dedup이 안전.
2. (권장) estSalary 상한 보정: 1인 고지액 상한 근접 행에 "상한 이상(≥7,400만)" 플래그/라벨로 과소추정 오해 방지. 점수 공식은 유지 가능.
3. (권장) 업종 표본 N<5(156개) 업종은 allMedian 또는 상위 업종군 중앙값 폴백으로 단일표본 노이즈 제거.
4. (선택) closed 풀 200 고정값 상수 분리 + 시도명 레거시/신규 정규화 매핑.
5. (선택) 잔존 8건(세종 sido + 타지역 jibun) 등 sido코드와 jibun 불일치 행 로깅.

## 반성점
- Round2 핵심 지적(통합시·세종 주소파싱)은 전수 재측정으로 완전 해소 확인(19.9%→0.00%, 세종 3,206→0). 수정 검증은 "수정 전 동일 지표를 다시 산출"하는 것이 결정적이었음.
- 새 발견은 dedup이었음: 표면상 191건 제거로 "안전"해 보였으나, bizNo가 유니크하다는 암묵 가정을 의심하고 충돌행을 첫행과 대조해 185건 과다제거를 잡음. 유니크값 32,241/552,693 측정이 결정타.
- turnover max=200·salary max=8,069 같은 "딱 떨어지는 상한"은 코드 캡과 데이터 자연 천장을 구분해야 함 — scoreCore에 캡이 없음을 확인하고 원본 군집(일용직 현장, 상한소득월액)으로 귀속시킴.
