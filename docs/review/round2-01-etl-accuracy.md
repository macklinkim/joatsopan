# Round2-01 · ETL / 실데이터 정확성 점검

대상: `scripts/etl.mjs`, `lib/score.ts`, `lib/data.ts`, `data/companies.json`, `data/raw/nps.csv`
데이터: 국민연금 가입 사업장 내역 2026-04, 원본 593,075행 / 활성 552,878곳 적재.

## 점검범위
- 컬럼 인덱스 매핑(`C` 객체) ↔ EUC-KR 헤더 정합성
- `estSalary` 공식·단위(만원)·상한소득월액 미반영 영향
- 지번주소 `split`(`sido/sigungu/dong`) 추출 견고성
- 업종 중앙값 산출(0 제외, 데이터 적은 업종 폴백)
- 휴폐업(status≠1) 처리
- `companies.json` 통계로 점수 분포·결측·이상치 실측 검증

## 발견 (심각도 · 근거)

### [심각] 1. 지번주소 파싱 오류 — 도(道) 내 통합시의 '동'이 통째로 누락 (활성 19.9% = 110,014곳)
- 근거: `etl.mjs:107` `const parts = (r.jibun||"").split(/\s+/)` 후 `parts[0]=sido, parts[1]=sigungu, parts[2]=dong` 고정 매핑.
- 통합시 주소는 4토큰 구조 "경기도 / 성남시 / 분당구 / 야탑동" 이다. 현재 코드는 sido=경기도, sigungu=성남시, **dong=분당구(구!)** 로 넣고 실제 동(야탑동)을 버린다.
- 원본 20만행 표본: 4토큰 38,112건(19%), `part[1]=시 & part[2]=구` 패턴 38,113건. 적재본 전수 측정 결과 `dong` 슬롯이 실제로 '구'로 끝나는 행 = **110,014 (19.9%)**. 예: `경기도 수원시 영통구`(dong=영통구), `경상북도 포항시 남구`, `경상남도 창원시 성산구`.
- 영향: 화면 표시 `addr`(`data.ts:71`)와 `dong` 값 오류, `nearbyCompanies`의 dong 티어(`data.ts:163`) 매칭이 "구 단위"로 부정확해짐. 다행히 1차 키인 `bdong`(법정동코드, col7)은 정상 보존되어 nearby 1순위는 동작.

### [심각] 2. 세종시 2토큰 주소 — 동/읍/면이 sigungu로 오적재 (3,222곳, dong 99.5% 결측)
- 근거: 세종시는 시군구 계층이 없어 원본이 "세종특별자치시(공백 다수)반곡동" 형태. `split(/\s+/)` → `["세종특별자치시","반곡동"]` 2토큰.
- 결과: `sigungu=반곡동`(동이 시군구로!), `dong=parts[2]||""` → 빈값. 적재본 세종 3,222행 중 3,206행 dong 결측 확인. `sigungu` 인터닝 리스트에 동/읍/면 33개가 오염됨(보람동, 전의면, 조치원읍…).

### [중] 3. estSalary 상한소득월액 미반영 — 고연봉 구간 체계적 과소추정·압축
- 공식 `estSalary=(notice/members/0.09)*12/10000` (`etl.mjs:11`, `score.ts:39`)는 단위(만원)·9%(근로자+사업주 합산 요율) 모두 타당.
- 그러나 연금료는 상한소득월액(2025년 약 617만원, 1인 고지 ≈ 555,300원)에서 캡된다. 실측: 적재 salary p99=7,383 / **max=8,069만원**, 이론 상한 ≈7,404만원(617만×12). 1인 고지액이 555,000~556,000원 구간에 몰린 행 183건 확인 → 캡 도달 증거.
- 영향: 실연봉 7,400만원↑ 기업은 모두 ~7,400만원으로 눌려 추정됨. 점수 s2(중앙값 대비 연봉)와 "박봉/안심 대상" 어워드에 상방 왜곡 없음(과소추정만). 분포 압축으로 고임금 변별력 상실. (전체 영향은 제한적이나 문서화 필요)

### [경미] 4. 업종 중앙값 — 설계 양호, 폴백 거의 무영향
- `salary>0`만 누적(`etl.mjs:60`), 0 제외 정상. 폴백 `indMedian||allMedian`(`etl.mjs:91,104`). 실측: 1,575개 업종 중 폴백(=allMedian 3,990)으로 떨어진 업종 단 2개. salary==0 적재행 0건. 데이터 적은 업종이라도 표본 1개면 그 값이 중앙값이 되는 잠재 노이즈는 있으나 현 분포상 미미.

### [경미] 5. 휴폐업 처리 — 의도대로 동작하나 99.4% 폐기
- status 코드 1=553,520 / 2(탈퇴)=39,555. ETL은 탈퇴 중 `members>=1`만 closedRows(245건) → 그중 상위 200건만 적재(`etl.mjs:56,125`). 탈퇴의 대부분(members<1)은 데이터가 비어 폐기되는 것이 맞으나, "별이 된 좋소" 풀이 200곳으로 고정됨은 명시 필요. status값 1/2만 존재(3 이상 없음).

### [확인 통과] 컬럼 매핑 / bizNo
- 헤더 대조: ym0 / name1 / **bizNo2** / status3 / jibun5 / bdong7 / sidoCd9 / indCode13 / indName14 / leaveDt17 / members18 / notice19 / hires20 / leaves21 — 전부 일치.
- bizNo는 전 행 길이=6 (사업자등록번호 앞 6자리 마스킹) 확인. `f.length!==22` 가드로 컬럼 어긋남 방지.

## 개선안
1. (필수) 주소 파싱을 토큰 고정 인덱스 대신 **접미사 규칙 기반**으로 교체: 마지막 토큰을 dong, 그 앞에서 끝이 `구`인 토큰을 sigungu에 병합(예 `성남시 분당구`)하거나, 더 견고하게는 이미 정확한 `bdong`(법정동코드 10자리)를 행정구역 사전과 조인해 sido/sigungu/dong을 도출. 세종시는 sigungu 빈값 허용.
2. (권장) estSalary에 상한 보정 주석/플래그 추가. 최소한 1인 고지액이 상한 근접 시 "상한 이상(≥7,400만)" 라벨로 표기해 과소추정 오해 방지. 점수 s2는 캡 영향이 위험도를 과대평가하지 않으므로 공식 유지 가능.
3. (선택) 업종 표본 N<5 업종은 allMedian 또는 상위 업종군 중앙값으로 폴백해 단일표본 노이즈 제거.
4. (선택) closed 풀 200 고정값을 상수로 분리하고 적재 로그에 "탈퇴 39,555 중 members≥1 245, 상위 200 적재" 명시.

## 반성점
- nearby가 `bdong` 1차 키 덕에 표면상 동작해, dong 표시값 오류(19.9%)가 가려져 있었음 — 표시 필드와 매칭 키를 분리 검증해야 함을 확인.
- estSalary는 공식만 보면 정확하나 도메인 제도(상한소득월액)를 모르면 "정확"으로 오판하기 쉬움 — 실측 분포(max 8,069, 캡 군집)로 교차검증한 것이 결정적.
- 세종시 2토큰·통합시 4토큰은 표본 상단(서울 위주)만 보면 놓치기 쉬움 — 전수 통계와 접미사 패턴 카운트로 잡아냄.
